# 天猫用户复购预测与用户增长分析：进阶版最终报告

## 1. 业务问题

大促后商家面临复购率低、全量触达成本高的问题。项目以用户—商家为粒度，为有限的营销资源识别高潜复购用户，并输出差异化触达建议。

## 2. 数据处理与 MySQL 分析

- 训练集：260,864 条用户—商家记录；复购率 6.11%。
- 行为日志：54,925,330 条；通过 Pandas 每次 100 万行分块聚合。
- 特征：行为频次、活跃间隔、用户对商家的行为占比、购买/加购转化、年龄段和性别等 23 项。
- MySQL：建立 `analysis_base` 表，使用 `CASE WHEN`、`GROUP BY`、`HAVING` 和 `COUNT(DISTINCT ...)` 完成行为分层、商家分析与数据质量校验。
- 数据质量：同一用户—商家重复训练样本数为 0。

## 3. 模型选择与离线评估

在相同的 80/20 分层验证集上比较逻辑回归和随机森林：

| 指标 | 逻辑回归 | 随机森林 |
| --- | ---: | ---: |
| ROC-AUC | 0.6127 | 0.5813 |
| PR-AUC | 0.1033 | 0.0896 |
| Top20% 用户复购率 | 10.61% | 9.47% |
| Recall@Top20% | 34.70% | 30.97% |
| Lift@Top20% | 1.74 | 1.55 |

逻辑回归相对随机森林的 AUC 差值 95% 配对自助法置信区间为 [0.0220, 0.0403]，因此选择逻辑回归作为最终复购排序模型。

## 4. 用户增长策略

按测试集复购得分 Top20% 划分高潜用户：

- 高潜加购用户（566 人）：加购商品降价提醒 + 限时优惠券；
- 高潜老客（51,730 人）：新品/补货提醒 + 老客专属权益；
- 低潜用户（209,181 人）：低成本触达或暂不投放。

Power BI 看板汇总模型效果、用户分层、行为复购率、商家榜单和模型对比，支持运营快速查看优先级。

## 5. 线上验证设计

离线模型结果不等于真实线上增长。上线时应将高潜人群随机拆分为实验组和对照组，观察 30 天增量复购率、增量 GMV 与 ROI，并通过双样本比例检验判断差异是否显著。
